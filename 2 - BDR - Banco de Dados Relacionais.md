<h1 align="center"> BDR - Bando de Dados Relacionais </h1>

Como apresentado no tópico anterior, sobre tipos de banco de dados, neste tópico vamos abordar o tipo de Bando de Dados Relacionais.

Um BDR é um tipo de banco de dados que organiza dados em tabelas, as quais são interligadas por **relacionamentos**. As **tabelas** são compostas por __linhas__ (registros) e __colunas__ (atributos). Essa estrutura facilita a visualização, o gerenciamento e a análise de informações complexas.

* Características Essenciais dos BDRs:
    * **Organização em Tabelas:** Cada tabela armazena dados relacionados a um único tema (entidade), como clientes, produtos ou pedidos.
    * **Relacionamentos entre Tabelas:** As tabelas se conectam através de chaves primárias e estrangeiras, permitindo a associação de dados entre diferentes entidades.
    * **Modelo Relacional:** Baseia-se na teoria matemática das relações, garantindo a consistência e integridade dos dados.
    * **Linguagem de Consulta:** A linguagem SQL (Structured Query Language) é utilizada para interagir com o banco de dados, realizar consultas, inserir, remover e atualizar dados.

* Vantagens dos BDRs:
    * **Estrutura Clara e Intuitiva:** A organização em tabelas torna a compreensão dos dados mais fácil, facilitando o acesso e a manipulação de informações.
    * **Altamente Escalável:** Podem armazenar grandes volumes de dados de forma eficiente, atendendo às necessidades de empresas em crescimento.
    * **Flexibilidade: Suportam** diversos tipos de dados, como números, textos, imagens e datas, permitindo o armazenamento de informações variadas.
    * **Segurança Robusta:** Oferecem mecanismos de autenticação, autorização e criptografia para proteger dados confidenciais contra acessos não autorizados.
    * **Padronização:** A linguagem SQL é um padrão internacional, facilitando a migração de dados entre diferentes sistemas.
 
  Neste estudo estarei utilizando o SGBD o MySQL. Ele é um SGBDR open-source popular, utilizado em diversos projetos web e de software.
