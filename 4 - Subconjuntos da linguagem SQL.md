<h1 align="center">  Subconjuntos SQL <h3> </h1>

A linguagem **SQL**, ou *Structured Query Language*, é composta por **cinco** subconjuntos principais, cada um com funções específicas para gerenciar dados em bancos de dados relacionais:


**1. Linguagem de Definição de Dados (DDL - Data Definition Language):**
   
    * Função: Criar, modificar e remover estruturas no banco de dados, como tabelas, colunas, índices e views.
      Comandos: CREATE, ALTER, DROP
---
**2. Linguagem de Manipulação de Dados (DML - Data Manipulation Language)**
   
    * Função: Inserir, atualizar, excluir e recuperar dados armazenados nas tabelas do banco de dados.
      Comandos: INSERT, UPDATE, DELETE, SELECT
---
**3. Linguagem de Controle de Dados (DCL - Data Control Language):**

    * Função: Gerenciar acessos e permissões de usuários no banco de dados, garantindo segurança e integridade das informações.
      Comandos: GRANT, REVOKE.
---
**4. Linguagem de Consulta de Dados (DQL - Data Query Language):**

    * Função: Extrair informações específicas do banco de dados através de consultas complexas, utilizando filtros, ordenações e agregações.
      Comandos: SELECT, combinados com cláusulas como FROM, WHERE, ORDER BY, GROUP BY, HAVING.
---
**5. Linguagem de Controle de Transações (TCL - Transaction Control Language):**

    * Função: Gerenciar transações no banco de dados, garantindo a consistência e a confiabilidade das operações.
      Comandos: BEGIN TRANSACTION, COMMIT, ROLLBACK
